JBoss Application Server

If this is your first time using XJAF, you need to download JBoss JBoss AS (ZIP archive, 110 MB). After unpacking the archive, please set an environment variable named JBOSS_HOME to point to the newly created folder
(e.g. C:\jboss-eap-6.1 on Windows, or /home/myself/jboss-eap-6.1 on Linux).

Please note that XJAF and JBoss AS require Java SE Development Kit (JDK) 7 to run. The Java Runtime Environment (JRE) will not suffice. You may download JDK 7 here. To make sure the correct Java version is used, you should set an environment variable JAVA_HOME to point to the JDK installation folder (e.g. C:\Program Files\Java\jdk1.7.0_55 on Windows, or /usr/lib/jvm/open-jdk on Linux).

Downloading and installing XJAF

Download the latest version of XJAF (ZIP archive). To install XJAF, follow the next set of commands:

  • Unpack the ZIP archive anywhere. Open the command prompt / terminal, go to the xjaf2x folder of your XJAF installation, and execute start-node.bat under Windows, or ./start-node.sh under Linux;
  • Wait until you see "JBoss EAP 6.1.0.GA (AS 7.2.0.Final-redhat-8) started in XYZms";
  • Navigate to the bin folder of your JBoss installation and type jboss-cli.bat --gui in Windows command prompt, or ./jboss-cli.sh --gui in Linux terminal;
  • CLI GUI window should pop-up. Choose Deployments -> Deploy from the main menu. Browse the xjaf2x.jar file in your XJAF installation, specify xjaf2x for the Name field, and mark the xjaf2x-group check box. Leave other fields empty / unchecked. Click OK to close the dialog, and finally Submit to deploy XJAF to JBoss.

Your XJAF installation is now ready. You may open the following address in your web browser to access the XJAF client: http://localhost:8080/xjaf2x. From there, you may explore the framework's functionalities and run some of the bundled example agents. For more information about configuring XJAF and starting a computer cluster, please visit the documentation page.
