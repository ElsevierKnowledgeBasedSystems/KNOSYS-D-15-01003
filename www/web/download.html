Download the latest version of XJAF (ZIP archive, ~130 MB).

The archive includes WildFly enterprise application server (formerly: JBoss), pre-configured to work with XJAF. After unpacking the archive, please set an environment variable named JBOSS_HOME to point to the wildfly-9.x sub-folder
(e.g. C:\siebog\wildfly-9.x on Windows, or /home/myself/siebog/wildfly-9.x on Linux).

Please note that XJAF and WildFly require Java SE Development Kit (JDK) 7 or later to run. The Java Runtime Environment (JRE) will not suffice. You may download the latest JDK here. To make sure the correct Java version is used, you should set an environment variable JAVA_HOME to point to the JDK installation folder
(e.g. C:\Program Files\Java\jdk1.8.0_1 on Windows, or /usr/lib/jvm/java-8-oracle on Linux).

Finally, to start XJAF, go to the unpacked XJAF folder, and execute start-node.bat under Windows, or start-node.sh in Linux terminal.

Your XJAF installation is now ready. You may open the following address in your web browser to access the XJAF client: http://localhost:8080/siebog. From there, you may explore the framework's functionalities and run some of the bundled example agents. For more information about configuring XJAF and starting a computer cluster, please visit the documentation page.
